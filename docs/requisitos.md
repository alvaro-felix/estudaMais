# Requisitos do Projeto – Estuda+

## Objetivo
Desenvolver um sistema web para auxiliar estudantes universitários na organização e acompanhamento de seus estudos, contribuindo para o ODS 4 – Educação de Qualidade.

## Problema
Muitos estudantes apresentam dificuldade em manter constância nos estudos, acompanhar seu próprio progresso e organizar melhor seu tempo acadêmico.

## Solução
O sistema Estuda+ permitirá o cadastro de disciplinas, definição de metas de estudo, registro de sessões de estudo e visualização de progresso acadêmico.

## Requisitos Funcionais
1. O sistema deve permitir o cadastro de usuários.
2. O sistema deve permitir login de usuários.
3. O sistema deve permitir o cadastro de disciplinas.
4. O sistema deve permitir definir metas de estudo por disciplina.
5. O sistema deve permitir registrar sessões de estudo com duração e data.
6. O sistema deve calcular automaticamente o progresso do aluno em relação às metas.
7. O sistema deve exibir histórico de estudos.
8. O sistema deve permitir edição de informações.
9. O sistema deve permitir exclusão de registros de estudo.
10. O sistema deve apresentar feedback sobre o desempenho do usuário.
11. O sistema deve permitir ao administrador gerenciar usuários.
12. O sistema deve permitir ao administrador visualizar dados gerais do sistema.

## Requisitos Não Funcionais
1. O sistema deve possuir interface simples e intuitiva.
2. O sistema deve garantir autenticação segura.
3. O sistema deve armazenar os dados em banco persistente.
4. O sistema deve ter tempo de resposta adequado.
5. O sistema deve ser modular e de fácil manutenção.
6. O sistema deve validar entradas de dados.
7. O sistema deve garantir consistência das informações registradas.
